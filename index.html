The Participative Application of the NB3 Framework

C. Arslan et al., 2025

Welcome to the Interactive Web Map

This tool streamlines the collaboration in applying the Participative Downscaling Method of the NB3 Framework to identify spatially-relevant restoration units.

Users can add, view, or edit polygons—the Nature-based Building Blocks (NB3)—at any user-specified restoration site.

The tool complements the framework's application within the EU H2020 REST-COAST project:

 ➜ Pre-identified Coastal Units at three coastal restoration pilots are embedded: load using 🌊 RC

 ➜ For existing Coastal Units, stakeholders can work collaboratively on associated data in 📁 the pilots' shared directory: e.g. 📊 the Venice Lagoon's Artificial Salt Marsh Reconstruction

 ➜ For new Coastal Units, collaborate on a template file in e.g. 📁 the Venice Lagoon's shared directory: 📊 The NB3 Instance Template

New users of this tool can refer to the following publications for background on the NB3 Framework:

 1. C. Arslan et al. (2024) — Deliverable in REST-COAST Project Repository.

 2. C. Arslan & J.M. van Loon-Steensma (2025) — Scientific Articile in Nature-Based Solutions.

 3. C. Arslan et al. (2026) — Scientific Article in Ocean & Coastal Management.

The tool also supports application beyond coastal landscapes and EU context:

 ➜ Peatland and wet forest (WATERLANDS, REST-COAST's sister project): load using 🐸 WL

 ➜ Blank map for any landscape context: load using 🌍 MAP

Users can also load locally stored NB3 shape files: load using 📂 DIR

NOTE: After creating a new NB3 instance on the map:

 1. Go to 📁 the shared directory,

 2. Copy the link to a template file, e.g. 📊 The NB3 Instance Template newUnit1,

 3. Couple this link to the new NB3 instance via right click → Edit NB3 Unit

 4. Collaborate on the file by identifying biophysical, ecological, and socio-economic data spatially relevant to this unit.

IMPORTANT: After any modification to NB3 Instances (metadata, shape, or new polygon):

 1.Use Save NB3 Units to store the updated JSON locally (file name includes a date stamp for version control).

 2.Upload to the shared project directory: upToDateNB3Units (contact cengiz.arslan@wur.nl for access.)
